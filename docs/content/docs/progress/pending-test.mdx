# 待测试

- Codex App 插件文档：新增插件安装教程，区分 AI 自动安装、手动安装和本仓库开发调试安装；同步优化插件 README 与技能说明。
- 配置与用户偏好：每个模型渠道新增 OpenAI / Gemini 调用格式选择，默认 OpenAI，切换默认格式时同步更新默认 Base URL；渠道 Tab 增加紧凑醒目的提醒，说明模型是否可选需要到“模型”Tab 选择。
- 配置与用户偏好：模型渠道中移除 GPT 图片构建和 GPT 图片修改的 `response_format` 下拉框，OpenAI 图片构建和图片修改请求固定使用 `b64_json`，返回结果优先读取 `b64_json`，没有时再读取 `url`。
- 图片加载：新增同源图片代理兜底，生成或修改接口返回无 CORS 头的远程图片 URL 时，保存到本地图片存储前会自动通过代理读取。
- 画布生图：新建画布生图和配置节点的默认生成张数从 3 张调整为 1 张。
- 部署缓存：移除页面入口禁用持久缓存的响应头，页面缓存策略交回 Next.js 与部署平台处理。
- 首页提示词展示：提示词接口响应缺少 `items`、`tags` 或 `categories` 时使用空数组兜底，避免首页渲染 `.map` 报错。
- 图片修改：调用 `/images/edits` 前会先把参考图片转换为 JPEG 文件再提交，蒙版仍保持原格式。
- Docker 镜像构建：为画布列表页补充 Suspense 边界，修复 Next 构建 `/canvas` 时 `useSearchParams` 导致的 prerender 失败。
- Docker 镜像构建：Next 构建阶段改用 Node 直接执行 `next build`，避免 `bun run build` 在部分构建环境中以 `exit code 132` 退出。
- Gemini 调用格式：支持拉取 Gemini 模型列表，并用于文本问答、在线 Agent 工具调用、基础生图和图生图请求。
- 音频和视频生成：选择 Gemini 调用格式时先给出不支持提示，仍需使用 OpenAI 兼容渠道。
